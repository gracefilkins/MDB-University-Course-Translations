Bonjour. Dans cette vidéo, vous découvrirez les index. Nous discuterons des indices, de la façon dont ils peuvent améliorer le rendement et des coûts de leur utilisation. Nous allons également examiner les types d'index les plus courants. 

Les index sont des structures de données spéciales qui stockait une petite partie des données de collections sous une forme ordonnée facile à parcourir et à rechercher efficacement. Les index pointaient vers l'identité du document et vous permettent de rechercher, d'accéder et de mettre à jour les données plus rapidement. Les index sont utilisés dans MongoDB pour améliorer les performances des requêtes. Ils peuvent accélérer les requêtes, réduire les entrées-sorties de disque et réduire les ressources requises pour ces requêtes. 

Les index prennent également en charge les requêtes telles que les correspondances d'égalité et les opérations basées sur la plage et renvoient les résultats triés. Les index stockconservent les données dans un format ordonné en fonction des zones d'index et de l'ordre de tri des valeurs qui sont fournis lors de la création de l'index. Sans index, MongoDB doit lire chaque document d'une collection en effectuant une analyse de collection pour vérifier s'il correspond à la requête en cours d'exécution. Il doit également trier les résultats en mémoire si la requête nécessite une sortie triée. 

Lors de l'utilisation d'un index, MongoDB extrait uniquement les documents identifiés par l'index en fonction de la requête et renvoie les résultats plus rapidement. Si l'index contient toutes les informations que la requête souhaite extraire, MongoDB n'a pas besoin de lire le document. Par défaut, un seul index est créé par collection. Cet index par défaut inclut uniquement la zone _id. Chaque requête doit utiliser un index. De cette façon, vous pouvez créer des index supplémentaires dans vos collections pour couvrir vos requêtes de données. 

Sachez que les index sont associés à un coût de performance en écriture. Lors de l'insertion de nouveaux documents dans la collection ou de la mise à jour des zones d'index, nous devons également mettre à jour les données dans la structure d'index. En outre, les performances d'écriture peuvent se dégrader si nous avons trop d'index dans une collection. Nous devons donc nous assurer que tous les index que nous avons sont utilisés. Sinon, nous devrions supprimer les index inutiles ou redondants. 

Maintenant que nous avons couvert les bases de l'indexation, allons-y sur les types d'index les plus courants dans MongoDB -- Des index de zone unique et des index composés. Les index de zone unique sont des index sur une seule zone. Les index composés incluent plusieurs zones de l'index. Dans ces deux index, les zones de début ou de préfixe d'index peuvent être utilisées pour prendre en charge les requêtes. Les deux types d'index peuvent également être des index multiclés s'ils fonctionnent sur une zone de tableau. Chaque entrée de tableau possède une entrée d'index correspondante. 

Allons à travers un exemple basé sur des requêtes exécutées sur la collection de clients. Voici un exemple de document de cette collection. La collection comprend des informations sur les clients, tels que le nom d'utilisateur, la date de naissance, s'ils sont actifs, les comptes qu'ils ont, ainsi que d'autres informations sur le compte. Disons que nous exécutons habituellement ces requêtes client pour rechercher tous les clients qui sont actifs et qui ont un compte spécifique. 

Dans ce cas, nous pouvons définir un index avec les champs actifs et comptes pour améliorer les performances de ces requêtes. Etant donné que l'index comporte deux zones et que l'un d'entre eux est un champ de tableau, il s'agit d'un index composé à plusieurs clés. Les deux requêtes peuvent utiliser cet index.
